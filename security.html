🔒 Security Features

All security enhancements implemented on this website — live and verifiable

Mandatory Security Implementations

ACTIVE

HTTPS + TLS 1.3

All traffic is encrypted via HTTPS with TLS 1.3. GitHub Pages automatically provisions and renews a free Let's Encrypt SSL certificate. Any HTTP request is permanently redirected (308) to HTTPS, enforcing encrypted connections for all visitors.

Protocol: TLS 1.3
Certificate: Let's Encrypt
HTTP → HTTPS: 308 Permanent Redirect
HSTS: max-age=31536000

Let's Encrypt · TLS 1.3 · HSTS · 308 Redirect

ACTIVE

XSS Protection via DOMPurify

Cross-Site Scripting (XSS) attacks inject malicious scripts into web pages. This site uses DOMPurify — the industry-standard JavaScript library — to sanitize all dynamic content before it is rendered in the DOM. URL hash parameters are sanitized on every page load.

// Live XSS sanitization demo:
const dirty = '<img src=x onerror=alert(1)>';
const clean = DOMPurify.sanitize(dirty);
// clean = '<img src=x>' — script removed

✅ ACTIVE

Content Security Policy (CSP)

A strict Content Security Policy is enforced via HTTP meta tags on every page. CSP tells the browser exactly which sources are allowed to load scripts, styles, and images — blocking any unauthorized injections even if XSS sanitization were bypassed.

default-src 'self';
script-src 'self' https://cdnjs.cloudflare.com
           https://www.googletagmanager.com;
style-src 'self' 'unsafe-inline';
img-src 'self' data: https://images.unsplash.com;
connect-src 'self' https://www.google-analytics.com;

CSP · Allowlist · Script injection · MIME types

Performance Optimizations

⚡

CDN Delivery

GitHub Pages serves content from a global edge network (CDN). Requests are routed to the nearest point of presence, reducing latency. DOMPurify is loaded from Cloudflare's CDN — one of the fastest global CDN providers.

CDN · Edge network · Latency · Anycast

⚡

Lazy Loading Images

All images on this site use the HTML5 loading="lazy" attribute. Images are only fetched by the browser when the user scrolls near them — saving bandwidth and improving initial page load time significantly.

<img src="image.jpg" loading="lazy" width="600" height="300">

Lazy loading · Bandwidth · Page speed · LCP

⚡

Minified Assets

CSS is minified (whitespace and comments removed) to reduce file size and transfer time. Minification is a standard production optimization that reduces payload size by 20–40%.

Minification · CSS optimization · Payload size

GitHub Actions CI/CD

A GitHub Actions workflow automatically validates the repository on every push to main. It checks all HTML files are present, scans for insecure HTTP references, and confirms deployment readiness — automating the deployment pipeline.
