Networking Concepts

Deep dive into DNS, IP addressing, HTTPS, TCP/IP, and how they all power this live website

Core Networking Concepts

01

DNS Resolution

The Domain Name System translates human-readable domain names into IP addresses. When you visit this site, your browser queries a DNS resolver which contacts the root nameserver → TLD nameserver → authoritative nameserver and finally gets back 185.199.108.153.

nslookup · dig · A records · TTL

02

IPv4 Addressing

IPv4 addresses are 32-bit identifiers written in dotted decimal. GitHub Pages uses four load-balanced anycast addresses, so your request goes to the nearest edge server automatically.

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153

32-bit · Dotted decimal · Anycast · Load balancing

03

IPv6 Addressing

IPv6 addresses are 128-bit identifiers providing vastly more address space than IPv4. This site supports dual-stack (both IPv4 and IPv6), future-proofing it for modern networks.

2606:50c0:8000::153
2606:50c0:8001::153
2606:50c0:8002::153
2606:50c0:8003::153

128-bit · Hexadecimal · Dual-stack · AAAA records

04

HTTPS Protocol

HTTPS is HTTP secured with TLS/SSL encryption. All communication to this site is encrypted end-to-end. HTTP requests are permanently redirected (308) to HTTPS. Let's Encrypt provides the free certificate.

TLS 1.3 · Let's Encrypt · 308 Redirect · HSTS

05

TCP/IP Model

The five-layer TCP/IP model describes how internet communication works from physical cables all the way up to your browser rendering this page.

LayerNameProtocols
5ApplicationHTTP/HTTPS, DNS, TLS
4TransportTCP, UDP
3InternetIPv4, IPv6, ICMP
2LinkEthernet, WiFi
1PhysicalCables, Fiber, Wireless

Five layers · Protocol stack · Encapsulation

06

Security Headers

HTTP response headers enforce security policies in browsers. This site implements HSTS, Content Security Policy, X-Frame-Options, and X-Content-Type-Options to protect against a range of attacks.

Strict-Transport-Security: max-age=31536000
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Content-Security-Policy: default-src 'self'

HSTS · CSP · XSS protection · Clickjacking

This website is hosted publicly on GitHub Pages. DNS resolves the domain to multiple load-balanced IP addresses. All communication is secured over HTTPS using TLS 1.3 within the TCP/IP protocol stack.
