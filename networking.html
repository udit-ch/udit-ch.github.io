Networking Concepts

Deep dive into DNS, IP addressing, HTTPS, and TCP/IP protocols

Core Networking Concepts

01

DNS Resolution

The Domain Name System translates human-readable domain names into IP addresses that computers use to locate servers on the internet.

nslookup · dig · A records

02

IPv4 Addressing

IPv4 addresses are 32-bit identifiers that uniquely identify devices on networks. GitHub Pages uses load-balanced anycast addresses.

32-bit · Dotted decimal · Anycast

03

IPv6 Addressing

IPv6 addresses are 128-bit identifiers that provide vastly more address space. Modern infrastructure uses dual-stack (both IPv4 and IPv6).

128-bit · Hexadecimal · Dual-stack

04

HTTPS Protocol

HTTPS is HTTP secured with TLS/SSL encryption. All communication is encrypted end-to-end. Let's Encrypt provides free certificates.

TLS 1.3 · Encryption · Let's Encrypt

05

TCP/IP Model

The five-layer TCP/IP model describes how internet communication works: Application → Transport → Internet → Link → Physical.

Five layers · Protocol stack

06

Security Headers

HTTP response headers enforce security policies in browsers. HSTS, X-Frame-Options, and CSP protect against web vulnerabilities.

HSTS · CSP · XSS protection

How This Website Works

DNS Resolution Process

When you visit this website:

  1. Browser queries a DNS resolver
  2. Resolver contacts root nameserver
  3. Root nameserver directs to TLD nameserver
  4. TLD nameserver directs to authoritative nameserver
  5. Authoritative nameserver returns IP address
  6. Browser connects to the IP address
