How Networking Makes This Site Work

01

DNS Resolution

DNS resolves udit-ch.github.io to GitHub's IPv4 addresses through the DNS hierarchy — root → TLD → authoritative nameserver.

nslookup · dig · A records

02

IP Addressing

GitHub Pages serves from four load-balanced IPv4 addresses and four IPv6 addresses using anycast routing for optimal performance.

IPv4 · IPv6 · Load balancing · Anycast

03

HTTPS / TLS 1.3

All traffic is served over HTTPS with TLS 1.3. HTTP requests return a 308 Permanent Redirect. HSTS enforced. Certificate by Let's Encrypt.

HTTP/2 · 200 OK · 308 Redirect · HSTS

04

Security Hardening

CSP headers block unauthorized scripts. DOMPurify sanitizes dynamic input. X-Frame-Options prevents clickjacking. X-Content-Type-Options stops MIME sniffing.

CSP · DOMPurify · XSS · Clickjacking

05

Performance & CDN

GitHub Pages edge network acts as a CDN. Images use lazy loading. CSS and JS are minified. Page load is optimized for speed and bandwidth efficiency.

CDN · Lazy loading · Minification

06

Traffic Monitoring

Google Analytics 4 tracks real-time visitor sessions, geographic distribution, and page performance — demonstrating network traffic analysis.

Google Analytics · Traffic · Monitoring
